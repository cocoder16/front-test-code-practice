구현 대상 - CRUD 투두리스트
포함 기술 - cypress, react-router, redux
나중에 - axios

* 아토믹 디자인 컴포넌트 설계

폼: 인풋, 서브밋 버튼
리스트 - 아이템: 내용(읽기 모드, 수정모드), 수정 버튼, 삭제 버튼

1: 버튼, 인풋, 텍스트
2: 아이템, 폼,
3: 리스트

* 리덕스 스토어 데이터 구조 설계

list: [
  {
    id,
    content,
  }
]

* 유닛 테스트 - 컴포넌트 렌더링 테스트, 상향식 통합

[ ] 버튼
[ ] 인풋
[ ] 텍스트
[ ] 아이템
[ ] 폼
[ ] 리스트
[ ] 페이지
[ ] 앱

* 유닛 테스트 - 이벤트 콜백 함수, 리듀서 함수

- 버튼
[ ] create 서브밋 함수
[ ] create 리듀서 함수
[ ] update 서브밋 함수
[ ] update 리듀서 함수
[ ] delete 서브밋 함수
[ ] delete 리듀서 함수
- 인풋
[ ] 인풋 체인지 함수

* BDD - 유저 시나리오 작성 -> 기능 테스트 작성 -> 테스트 통과 (axios, redux가 포함)

- 인풋을 작성하고 추가 버튼을 누른다
[ ] 리스트 더미 데이터 준비 (준비)
[ ] 셋업 - 리스트와 폼 = 앱 : props로 받아 렌더링된 상태 (준비)
[ ] 작성과 서브밋 이벤트 발생 (실행)
[ ] 리스트 목록 확인 (단언)
[ ] 인풋 클리어 확인 (단언)

- 수정 버튼을 누르고 수정하고 완료 버튼을 누른다
[ ] 위 셋업 이용
[ ] 수정 버튼 클릭, 작성, 서브밋 이벤트 발생 (실행)
[ ] 리스트 목록 확인 (단언)
[ ] 수정 모드가 사라지고 읽기 모드인지 확인 (단언)

- 삭제 버튼을 누른다
[ ] 위 셋업 이용
[ ] 삭제 버튼 클릭 이벤트 발생 (실행)
[ ] 리스트 목록 확인 (단언)
